ワークスペース内のプロジェクトを「レーン」として管理し、コンテキスト単位で一括切り替えできる VS Code 拡張機能。
複数プロジェクトで AI コーディングエージェントを並列運用する開発者を想定している。

## 課題

多数のプロジェクトを並行運用すると、以下の問題が起きる。

- エクスプローラーに全プロジェクトがフラットに並び、視認性が悪い
- ターミナルが全プロジェクト分混在し、切り替えが面倒
- エディタタブが全プロジェクト分開かれ、コードが混ざる
- 各プロジェクトでエージェントを並列実行している場合、どのプロジェクトで待機中かわからない

## 機能

各プロジェクトを「レーン」として扱い、エクスプローラー・エディタタブ・ターミナル・Git を一括で切り替える。
ワークスペースフォルダの実体は `.lanes-root/active` という symlink で、切替時にこの symlink の参照先だけが書き換わる。URI 自体は変わらないため拡張ホストが再起動せず、バックグラウンドのターミナルプロセスはそのまま生存し続ける。

- レーン切替
  - エクスプローラーと Git はアクティブレーンのみ表示
  - エディタタブをレーンごとに保存・復元
  - ターミナルは切替後もバックグラウンドで生存
- エージェント監視
  - Claude Code のプロセスを検出し idle/active をレーンごとに表示

## コマンド

| コマンド                         | 説明                                           |
| -------------------------------- | ---------------------------------------------- |
| `Project Lanes: Focus`           | レーンに切替                                   |
| `Project Lanes: Close Terminals` | アクティブレーンの全ターミナルセッションを終了 |

## 設定

| 設定                               | デフォルト | 説明                                           |
| ---------------------------------- | ---------- | ---------------------------------------------- |
| `projectLanes.refreshInterval`     | `1`        | エージェント状態のポーリング間隔（秒）         |
| `projectLanes.agent.idleThreshold` | `5`        | idle 判定までの無活動猶予（秒）                |
| `projectLanes.agent.showStatus`    | `true`     | バッジ・デコレーション・ステータスバーへの表示 |
| `projectLanes.terminal.shellPath`  | `""`       | Lane Terminal のシェルパス（空欄で `$SHELL`）  |

## 制約事項

- エージェント監視は Linux 専用（`/proc` を使用）。symlink ベースの構成も Linux 前提
- タブの復元は通常のファイルタブのみ（差分ビューやノートブック等は対象外）
- ターミナルセッションは VS Code ウィンドウのリロードでは保持されない
- `.code-workspace` の隣に `.lanes-root/` が作成される。必要に応じて `.gitignore` に追加すること
